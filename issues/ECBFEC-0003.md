## Problem Statement

There could be different templates for different types (e.g., bugs, features, enhancements).
The current template is intended for bugs.

## Proposed Solution

* Bug Report
  * Summary
  * Steps to Reproduce
    * Reproducibility(always/sometimes/rarely)
  * Expected Behavior
  * Actual Behavior
  * Severity/Priority
  * Environment
  * Logs/Screenshots
  * Additional Context

* Feature Request(to new functionality)
  * Problem Statement
  * Proposed Solution
  * Alternatives Considered
  * Who Benefits
  * Breaking Changes
  * Additional Context

* Enhancement(to new existing functionality)
  * Area to Improve
  * Current Behavior
  * Suggested Improvement
  * Expected Benefit
  * Notes

* Documentation
  * Issue type(Missing/Incorrect/Unclear/Outdated)
  * Location
  * Suggested Change
  * Additional Context(Why this matters)

* Task/Chore 
  * Task Description
  * Motivation
  * Acceptance Criteria
  * Dependencies
